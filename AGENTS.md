# Enterprise Website Redesign Specification

## Objective

Необходимо выполнить профессиональную модернизацию существующего корпоративного сайта.

Конечная цель — создать современный, производительный, доступный, SEO-оптимизированный многостраничный сайт на **Next.js + TypeScript**, используя существующий сайт только как источник информации, а предоставленный макет **Figma (через MCP)** — как основу визуального дизайна.

Копирование существующего сайта или макета недопустимо.

Необходимо создать значительно более качественный продукт, соответствующий современным требованиям UX/UI, SEO, производительности и архитектуры.

---

# Team

Работай как единая команда специалистов с опытом более 20 лет.

Команда включает:

- UX Designer
- UI Designer
- Product Manager
- Digital Marketing Manager
- SEO Specialist
- Accessibility Specialist
- Frontend Architect
- Senior Next.js Developer
- Performance Engineer
- Technical Lead

Все решения должны приниматься только после согласования всеми участниками команды.

При наличии противоречий необходимо выбирать решение, которое дает максимальную пользу конечному пользователю и бизнесу.

# Stage 2. Figma Analysis

Используя MCP, проанализировать выбранный макет Figma.

Не копировать дизайн буквально.

Использовать макет исключительно как источник:

- дизайн-системы
- сетки
- композиции
- типографики
- размеров
- spacing
- визуальной иерархии
- современных UI-паттернов
- компонентов
- анимаций
- принципов построения интерфейса

Если отдельные решения противоречат результатам Stage 1, использовать решение команды специалистов.

Приоритет всегда имеет качество продукта, а не точное соответствие макету.

# Expected Result

Итогом работы должен стать современный production-ready корпоративный сайт, который:

- значительно превосходит существующую версию;
- использует лучшие современные практики;
- обладает высокой производительностью;
- соответствует современным требованиям UX/UI;
- соответствует WCAG 2.2 AA;
- максимально оптимизирован для SEO;
- имеет масштабируемую архитектуру;
- готов к дальнейшему развитию;
- легко поддерживается;
- не является копией существующего сайта или макета Figma, а представляет собой самостоятельный современный продукт, созданный на основе профессионального анализа.
